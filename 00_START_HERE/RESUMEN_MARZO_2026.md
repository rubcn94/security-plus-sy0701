#### 1. Incident Response (6 Fases)
```
1. Preparation
2. Identification
3. Containment
4. Eradication
5. Recovery
6. Lessons Learned
```

#### 2. Orden de Volatilidad (Forensics)
```
1. CPU, caché
2. RAM
3. Estado de red
4. Procesos
5. Archivos temporales
6. Disco duro
7. Logs remotos
8. Backups
```

#### 3. Fórmulas Risk Management
```
SLE = Asset Value × Exposure Factor
ALE = SLE × ARO
ROI = (ALE_before - ALE_after - Cost) / Cost × 100%
```

#### 4. Subnetting Rápido
```
/24 = 254 hosts
/25 = 126 hosts
/26 = 62 hosts
/27 = 30 hosts
/28 = 14 hosts
/29 = 6 hosts
/30 = 2 hosts (point-to-point)
```

#### 5. Windows Event IDs
```
4624 = Logon exitoso
4625 = Logon fallido
4672 = Privilegios especiales
4688 = Proceso creado
4720 = Usuario creado
4732 = Usuario añadido a grupo
4740 = Cuenta bloqueada
4768 = Kerberos TGT solicitado
4769 = Kerberos Service Ticket
```

#### 6. Ports Comunes (PBQs)
```
20/21:  FTP
22:     SSH
23:     Telnet
25:     SMTP
53:     DNS
80:     HTTP
110:    POP3
143:    IMAP
443:    HTTPS
445:    SMB
3389:   RDP
```

#### 7. CIA Triad
```
Confidentiality = Encryption, Access Control, MFA
Integrity       = Hashing, Digital Signatures, Checksums
Availability    = Backups, RAID, Redundancy, UPS
```
